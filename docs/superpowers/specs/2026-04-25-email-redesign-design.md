# Email Redesign — Design Spec

## Goal

Replace the sparse, text-only portfolio alert email with a visually polished, information-rich notification that shows recent returns, current holdings, and trades — presented in a layout that adapts to the user's system light/dark mode preference.

---

## Build Pipeline

MJML is used to author email templates. The compiled HTML is committed to git so Go builds have no Node.js dependency.

```
alert/email/
  assets/
    pv-icon-blue.jpg       ← original copied from frontend-ng/public/
    logo-80.jpg            ← resized 80×80, generated by Makefile
  templates/
    src/
      success.mjml
      failure.mjml
      package.json         ← mjml dev dependency (pinned version)
    success.html           ← compiled, committed to git
    failure.html
  template.go              ← embed + render logic (minor extensions)
  email.go                 ← send logic (unchanged)
```

**Makefile target (`email-templates`):**
1. `sips` resizes `pv-icon-blue.jpg` → `logo-80.jpg` (80×80)
2. `npm install` in `templates/src/`
3. `npx mjml` compiles each `.mjml` → corresponding `.html`

The logo is embedded in the Go binary via `//go:embed assets/logo-80.jpg`. A package-level `var logoDataURL string` is computed at `init()` time as `data:image/jpeg;base64,<bytes>`. It is injected into `Payload.LogoDataURL` at send time.

## Visual Design

### Aesthetic

Professional financial dashboard aesthetic. The brand accent color is pulled from the icon's teal background (`#0ea5e9`). Typography is system-safe (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`). Generous whitespace, clear hierarchy, crisp tables.

### Light Mode (default / Gmail fallback)

- Background: `#f1f5f9` (slate-100)
- Card background: `#ffffff`
- Primary text: `#0f172a` (slate-900)
- Secondary text: `#64748b` (slate-500)
- Border: `#e2e8f0` (slate-200)
- Positive return: `#16a34a` (green-700)
- Negative return: `#dc2626` (red-600)
- Accent / header: `#0ea5e9` (sky-500, matches icon)

### Dark Mode (`@media (prefers-color-scheme: dark)`)

- Background: `#0f172a` (slate-950)
- Card background: `#1e293b` (slate-800)
- Primary text: `#f1f5f9` (slate-100)
- Secondary text: `#94a3b8` (slate-400)
- Border: `#334155` (slate-700)
- Positive return: `#4ade80` (green-400)
- Negative return: `#f87171` (red-400)
- Accent / header: `#38bdf8` (sky-400, slightly lighter for dark bg)

The header uses the accent color as a left border or subtle top stripe (not a full dark background block) so it reads well in both modes without fighting the `prefers-color-scheme` media query. The icon sits left-aligned next to "Penny Vault" in brand text. Portfolio name is right-aligned, large, in primary text color.

The returns grid uses a 5-column `mj-section`/`mj-column` layout. Each cell has a muted label row and a bold colored value row. Cells are separated by thin vertical dividers.

Tables (trades, holdings) use alternating row shading (`#f8fafc` / `#ffffff` in light, `#1e293b` / `#273448` in dark) and a bold header row in the accent color.

## Data Pipeline

### Payload struct additions

```go
// Logo
LogoDataURL template.URL

// Returns
DayChangePct   string  // e.g. "+0.42%"
DayChangeColor string  // "#16a34a" or "#dc2626"
WtdPct         string
WtdColor       string
MtdPct         string
MtdColor       string
YtdPct         string  // already computed, promote from Kpis
YtdColor       string
OneYearPct     string  // already computed, promote from Kpis
OneYearColor   string

// HoldingRow — add Shares field
type HoldingRow struct {
    Ticker    string
    Shares    string  // NEW: formatted with commas; "—" for $CASH
    Value     string
    WeightPct string
}
```

### Returns calculation

A new `buildReturns(ctx, db)` helper queries `perf_data` ordered by date DESC to get the full series, then:

- **Day**: `(last / prev) - 1`
- **WTD**: `(last / first row on or after most recent Monday) - 1`
- **MTD**: `(last / first row on or after 1st of current month) - 1`
- **YTD**: already in `Kpis.YtdReturn`
- **1Y**: already in `Kpis.OneYearReturn`

All use the closest available trading day when the exact calendar date has no row.

### Holdings calculation

`snapshot.Reader.CurrentHoldings()` already returns per-ticker quantity and market value. `WeightPct` is derived as `marketValue / total * 100`. `Shares` comes from `quantity`. `$CASH` renders `Shares` as `—`.

## Portfolio Link

A "View Portfolio" CTA button appears at the bottom of the body, above the footer. It links to the portfolio's page in the Penny Vault web app.

`Payload` gains a `PortfolioURL template.URL` field. The app base URL is supplied via the existing server config (a new `--app-base-url` flag, default `https://www.pennyvault.com`). The link is constructed as `{appBaseURL}/portfolios/{slug}`.

```
├─────────────────────────────────────────────┤
│  [View Portfolio →]  (accent-colored button) │
├─────────────────────────────────────────────┤
│ FOOTER                                      │
```

If `PortfolioURL` is empty the button is omitted (graceful degradation for on-demand emails where no base URL is configured).

## Unsubscribe

A low-prominence "Unsubscribe" text link appears in the footer below the View Portfolio button (standard placement, per email convention).

### Token

The unsubscribe URL is `{appBaseURL}/api/alerts/unsubscribe?token=<token>` where `token` is an HMAC-SHA256 signature over `alertID:recipientEmail` using a server secret (`--unsubscribe-secret` flag). This prevents anyone from unsubscribing an address they don't control.

`Payload` gains an `UnsubscribeURL template.URL` field. It is populated per-recipient at send time.

### Endpoint

A new unauthenticated `GET /api/alerts/unsubscribe` endpoint:
1. Verifies the HMAC token
2. Removes the recipient from the alert's `recipients` array
3. If the recipients array becomes empty, deletes the alert entirely
4. Returns a plain HTML confirmation page ("You have been unsubscribed from alerts for {portfolio name}")

The endpoint lives in the `alert` package alongside the existing handler. It requires no auth middleware since the token is the credential.

On-demand `SendSummary` emails omit the unsubscribe link (no persistent alert record to unsubscribe from).

## Scope Boundaries

- Failure email redesign is included (same header, error card replaces body).
- Plaintext version is updated to match new content fields.
- No changes to alert frequency logic, send infrastructure, or API endpoints.
- The on-demand `SendSummary` endpoint reuses the same template (trades section already suppressed there via empty `Trades` slice).
